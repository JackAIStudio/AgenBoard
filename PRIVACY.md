# 隐私说明

AgenBoard 不包含广告、用户画像、跨 App 跟踪或第三方分析 SDK。项目维护者不运营后端服务器、账号系统、录音中转、云存储或数据收集服务。正常使用 AgenBoard 时，录音、转写文本、热词和阿里云 API Key 不会发送给项目维护者，因此项目维护者没有接收、检索或查看这些内容的入口。只有用户主动通过 Issue、邮件或其他反馈渠道提交内容时，接收方才会看到用户提交的信息。

开源使当前实现可以被独立审查，但开源本身不替代隐私边界。本说明适用于本仓库当前源代码；第三方修改、派生或重新分发的版本可能采用不同的数据处理方式。

## 数据如何处理

- 每次录音及其识别历史默认保存在当前设备。用户在 App 中删除对应历史时，本地录音也会被删除。
- 使用 Apple 识别时，录音和转写由系统语音能力处理；iOS 26 使用设备端 SpeechAnalyzer，iOS 17–25 使用 Apple Speech 兼容路径，是否联网由 Apple 系统和设备能力决定。两条路径都不经过项目维护者的服务器。
- 只有用户主动选择阿里云识别方案后，主 App 才会使用用户自己的 API Key，将本次录音和当前启用的热词直接发送到阿里云百炼（DashScope）。文件版将录音上传至百炼托管的私有临时存储，与用户的阿里云主账号和 `fun-asr` 模型绑定，并在 48 小时后自动清理；实时版在录音过程中通过 WebSocket 将音频流直接发送给 `fun-asr-realtime`，不使用临时文件 URL。项目维护者不能通过 AgenBoard 访问这些数据；用户无需另行开通 OSS。数据处理受阿里云相应服务条款和隐私政策约束。
- 阿里云 API Key 保存在设备钥匙串。当前采用 BYOK（用户自带 API Key）模式，App 直接访问阿里云，项目方不提供中转服务器；如果未来改为由项目方统一提供托管识别服务，则应由可信服务端管理项目方凭证。
- 热词、快捷短语、设置和 Rime 用户词典默认保存在设备本地或 App Group 容器中。
- 键盘扩展自身不实现网络请求。开启“允许完全访问”是为了让键盘通过 App Group 与主 App 共享语音状态、识别结果、设置和快捷短语。

## 导入与导出

用户可以主动导出数据 ZIP。拼音学习数据会以 Rime 原生、可读的 `pinyin/rime_ice.userdb.txt` 文本快照随用户数据导出，其中包含用户输入过的候选、拼音编码、使用次数、权重和时间数据。原始录音和 API Key 默认不会导出；只有用户分别明确开启对应选项时才会加入数据包。包含 API Key 的 `credentials.json` 是明文文件，应像密码一样妥善保管。

## 删除与控制

用户可以在 App 中删除识别历史、录音、热词、快捷短语和已保存的阿里云配置。卸载主 App 会移除其本地容器；系统对 App Group 和钥匙串数据的保留行为可能与普通容器不同，如需彻底清理请先在 App 内删除相关内容。

## Apple 隐私清单

主 App 和键盘扩展均包含 `PrivacyInfo.xcprivacy`。清单按当前实现声明：无跟踪；阿里云模式可能发送音频和用户内容用于 App 功能；同时声明项目实际使用的 UserDefaults、文件元数据和系统启动时间类 API 的用途。
